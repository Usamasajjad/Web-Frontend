CSS Transitions

CSS transitions allows you to change property values smoothly, over a given duration. The following are common transition properties:
1. transition: A shorthand property for setting the four transition properties into a single property
2. transition-delay: Specifies a delay (in seconds) for the transition effect
3. transition-duration: Specifies how many seconds or milliseconds a transition effect takes to complete
4. transition-property: Specifies the name of the CSS property the transition effect is for
5. transition-timing-function: Specifies the speed curve of the transition effect

To create a transition effect, you must specify two things:
a. the CSS property you want to add an effect to
b. the duration of the effect

CSS Animations

CSS allows animation of HTML elements without using JavaScript. An animation lets an element gradually change from one style to another. To use CSS animation, you must first specify some keyframes for the animation. Keyframes hold what styles the element will have at certain times.
The following are common declarations for animations:
1. @keyframes: specifies the animation code. legal values are ---> from{} -- to{} -- 0%{} to 100%{}.
2. animation-name: specifies a name to bind the keyframe to the element.
3. animation-duration: defines how long an animation should take to complete.
4. animation-delay: specifies a delay for the start of an animation.
5. animation-iteration-count: specifies the number of times an animation should run.
6. animation-direction: specifies whether an animation should be played forwards, backwards or in alternate cycles.
7. animation-timing-function: specifies the speed curve of the animation.
8. animation-fill-mode: specifies a style for the target element when the animation is not playing.
9. animation: shorthand property. order ---> name -- duration -- timing-function -- delay -- iteration-count -- direction.

CSS Media Queries

Media queries allow us to change the layout of the page or the properties of elements upon broswer resizing. This allows us to deliver a better more personalized experience on all devices. An example of media queries would be: Using a 6x6 grid on desktop but rearraging it to 2x2x2x2x2 for mobile.

@media print {} is used to define how the page will look if it is printed.
@media screen and (min-width: 300px) {} is used to define how the page will look for different widths. The properties we can use are min-width/max-width. The connector we can use are and | or | not. We can also target different device orientations for example: @media screen and (orientation: landscape) and (min-width: 300px).

Example: //change background to black for devices with min width of 300 or greater. 

        @media screen and (min-width: 300px) {
            body {
                background: black;
            }
        }

CSS Variables

CSS Variales are mostly used to define styles only once as variables. Then use those variables through out the document. For example: background: var(--mycolor); is easier to define than background: 2DAD2D;. CSS variables have access to the DOM, which means that you can create variables with local or global scope, change the variables with JavaScript, and change the variables based on media queries. Best practice is to define them in the :root of the CSS.
Example: //This is how we make and use variables. 

               :root {
                   --mycolor: blue;
               }
               #logo {
                   background: var(--mycolor);
               }
