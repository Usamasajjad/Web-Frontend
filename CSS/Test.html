CSS Cascades, Specificity and Inheritance:

Concept Description
Cascades The order of CSS rules matters, when two rules apply that have equal specificity the one that comes last in the CSS is the one that will be used. For example: If two h1 selectors with different colors are specified, the one which comes last in the source order will be used.
Specificity If there are two or more conflicting CSS rules that point to the same element, the browser follows some rules to determine which one is most specific and therefore wins out. It is easier to check this by doing the following calculation. For each CSS declaration,
• If the element has inline styling, count 1000
• For each ID value, count 100
• For each class value count 10
• For each element reference count 1
The declaration with the highest count wins and is most specific therefore it will be applied.
Inheritance Inheritance is a process of receiving values of properties by a child element from its parent element. For example, If I declare a 'p' element, and italicize 'i' some part of it, then apply some css styling to the 'p' element such as color, font, etc. The styling is also applied to the 'i' element, which is a child of 'p'.

Common CSS Concepts/Properties

CSS Background

We can specify common background properties in a single line such as:
background: #ffffff url("img_tree.png") no-repeat right top;
The order of specifying properties is:
1. background-color
2. background-image
3. background-repeat
4. background-attachment
5. background-position
It doesn't matter if a certain value is missing as long as propertiesare defined in order.

CSS Borders

We can specify common border properties in a single line such as:
border-left: 6px solid red;
The order of specifying properties is:
1. border-width
2. border-style (required)
3. border-color
To make corners rounded, use border-radius property.

CSS Margins

The margin properties are used to create space around/outside of elements. We can specify margins for each side of the element, i.e top/bottom/left/right. However, We can also specify all of them in a single line, i.e margin: 50px 50px 50px 50px;
We can also assign the value 'auto' to the margin property to have the element centered.

IMPORTANT: The top and bottom margins of elements are sometimes collapsed into a single margin that is equal to the largest of the two margins. For example if two adjacent elements, h1 and h2 both have a margin value of 50, the margin between them will be 50 and not 100. This does not happen on left and right margins, Only on top and bottom margins.
It can be avoided by adding a simple empty 'p' element in between.

CSS Padding

The padding properties are used to generate space around/inside an element's content. We can specify paddings for each side of the element, i.e top/bottom/left/right. However, We can also specify all of them in a single line, i.e padding: 5px 10px 5px 10px;

IMPORTANT: Padding adds to the total width of the element. This can be undesirable if the element has a widht that is not 100%. To avoid this, use box-sizing: border-box;.

CSS Box Model

All HTML elements can be considered as boxes. In CSS, the term "box model" is used when talking about design and layout. The CSS box model is essentially a box that wraps around every HTML element. It consists of: margins, borders, padding, and the actual content. In that order! Further description of the model:
Content - The content of the box, where text and images appear
Padding - Clears an area around the content. The padding is transparent
Border - A border that goes around the padding and content
Margin - Clears an area outside the border. The margin is transparent

IMPORTANT: In order to set the width and height of an element correctly in all browsers, The total width of an element should be calculated in the following manner:
Total element width = width + left padding + right padding + left border + right border + left margin + right margin
The total height of an element should be calculated in the following manner:
Total element height = height + top padding + bottom padding + top border + bottom border + top margin + bottom margin

CSS Outlines

An outline is a line that is drawn around elements, OUTSIDE the borders, to make the element "stand out". Outline differs from borders. Unlike a border, the outline is drawn outside the element's border, and may overlap other content. Also, the outline is NOT a part of the element's dimensions; the element's total width and height is not affected by the width of the outline. We can specify multiple outline properties in a single line i.e outline: 5px solid yellow;. The order is:
1. outline-width
2. outline-style (required)
3. outline-color We can also add space between the border and the outline using the 'outline-offset' property.

CSS Text

Common text declarations:
1. color: blue; --- change text color.
2. text-align: center/right/left/justify --- align text.
3. text-decoration: none/overline/underline/line-through --- basic text decorations.
4. text-transform: lowercase/uppercase/capitalize --- change text case.
5. text-indent: 50px --- specify indent of first line of a para.
6. letter-spacing: 5px --- specify space between letters.
7. line-height: 2px --- specify space between lines.
8. word-spacing: 5px --- specify space between individual words.
9. text-shadow: 2px 2px 5px red; --- give shadow effect to text. Format: (horizontal)px (vertical)px (blur)px (color).

CSS Fonts

Common font declarations:
1. font-family: "Times New Roman", Times, serif; --- specify font family.
2. font-size: px/%/large --- specify font size.
3. font-style: normal/italic --- specify font style.
4. font-variant: normal/smallcaps --- specify font variant.
5. font-weight: normal/bold --- specify font weight.
6. link rel="stylesheet" href="https://fonts.googleapis.com/css?family=Audiowide|Sofia|Trirong" --- link external fonts.

CSS Links

Common link styling declarations:
1. a:link --- for unvisited links.
2. a:visited --- for visited links.
3. a:hover --- for link hover.
4. a:active --- for selecting/clicking on the link.

CSS Lists

Common List styling declarations:
1. list-style-type: circle/square/roman/none/etc --- specify item marker.
2. list-style-image: url("abc.png") --- use an image as an item marker.
3. list-style-position: inside/outside --- specify item marker position.

CSS Tables

Common Table styling declarations:
1. border: 1px solid black --- specify table border.
2. border-collapse: collapse --- table and th/td have separate borders. use this to remove double borders.
3. widht/height --- use these properties to manipulate column/row heights.
4. vertical-align/text-align --- use these properties to style text alignment.
5. tr:hover --- use to style table rows on hover.

CSS Display Property

The display property specifies if/how an element is displayed. Every HTML element has a default display value depending on what type of element it is. The default display value for most elements is block or inline.
A block-level element always starts on a new line and takes up the full width available (stretches out to the left and right as far as it can).
An inline element does not start on a new line and only takes up as much width as necessary.
Display: Inline; --- make an element behave as inline.
Display: Block; --- make and element behave as block.
Tip: Use max-width instead of width to better handle block elements upon resizing.

CSS Position Property

The position property specifies the type of positioning method used for an element. There are five different position values:
1. Static --- it is default for all html elements and not positioned in any special way.
2. Relative --- positioned relative to its normal position. must set right/left/top/bottom = px value.
3. Fixed --- always stays in the same place even if the page is scrolled like 'scroll to top'.
4. Absolute --- is positioned relative to the nearest positioned ancestor.
5. Sticky --- is positioned based on the user's scroll position.
Tip: z-index: -1 or 1; allows us to overlap elements the way we want.

CSS Overflow Property

The overflow property specifies whether to clip the content or to add scrollbars when the content of an element is too big to fit in the specified area.
1. overflow: visible; --- content isnt clipped and is visible outside the box.
2. overflow: hidden; --- content is clipped and hidden outside the box.
3. overflow: scroll; --- content is clipped and scrollbar appears.
4. overflow; auto; --- similar to scroll but adds scrollbar only when necessary.
5. overflow-x/overflow-y --- assign above values to specific axis. i.e enable horizontal scroll only.

CSS Float Property

The float property is used for positioning and formatting content e.g. let an image float left to the text in a container.
1. float: right; --- The element floats to the right of its container.
2. float: left; --- The element floats to the left of its container.
3. float: none; --- The element does not float.
The clear property specifies what elements can float beside the cleared element and on which side.
1. clear: none; --- allows floating elements Grid of Boxes / Equal Width Boxeson both sides.
2. clear: both; --- no floating elements allowed on either side.
3. clear: left; --- no floating elements allowed on the left side.
4. clear: right; --- no floating elements allowed on the right side.
The most common way to use the clear property is after you have used a float property on an element.

CSS Inline Block

Compared to display: inline, the major difference is that display: inline-block allows us to set width and height of the element. Also, with display: inline-block, the top and bottom margins/paddings are respected. Compared to display: block, the major difference is that display: inline-block does not add a line-break after the element, so the element can sit next to other elements.
