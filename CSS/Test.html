CSS Cascades, Specificity and Inheritance:

Concept Description
Cascades The order of CSS rules matters, when two rules apply that have equal specificity the one that comes last in the CSS is the one that will be used. For example: If two h1 selectors with different colors are specified, the one which comes last in the source order will be used.
Specificity If there are two or more conflicting CSS rules that point to the same element, the browser follows some rules to determine which one is most specific and therefore wins out. It is easier to check this by doing the following calculation. For each CSS declaration,
• If the element has inline styling, count 1000
• For each ID value, count 100
• For each class value count 10
• For each element reference count 1
The declaration with the highest count wins and is most specific therefore it will be applied.
Inheritance Inheritance is a process of receiving values of properties by a child element from its parent element. For example, If I declare a 'p' element, and italicize 'i' some part of it, then apply some css styling to the 'p' element such as color, font, etc. The styling is also applied to the 'i' element, which is a child of 'p'.
